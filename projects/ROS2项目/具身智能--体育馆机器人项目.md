## 3.nav2切换地图

  你这个项目里，Nav2 地图是在启动文件 my_navigation2/launch/navigation2.launch.py:19 里通过 map 参数
  传进去的，默认值现在是：

```json
  map_yaml_path = launch.substitutions.LaunchConfiguration(
      'map', default=os.path.join(my_navigation2_dir, 'maps', 'iot1.yaml'))
```

  所以有两种用法。

  第一种：临时切换，不改文件，直接启动时指定地图

```bash
cd mynav2/
source install/setup.bash
ros2 launch my_navigation2 navigation2.launch.py map:=/home/chuiyu/mynav2/src/my_navigation2/maps/
你的地图.yaml
```

  第二种：修改默认地图，改这个文件：
```bash
my_navigation2/launch/navigation2.launch.py:19
```

  把默认的：

```python
default=os.path.join(my_navigation2_dir, 'maps', 'iot1.yaml')
```

  改成：

```python
default=os.path.join(my_navigation2_dir, 'maps', '你的地图.yaml')
```

  另外，如果你是新建一张地图，要确认地图 yaml 文件本身也正确，比如现在的 my_navigation2/maps/
  iot1.yaml:1 里第一行是图片路径：

```python
image: /home/chuiyu/mynav2/src/my_navigation2/maps/iot1.pgm
```

  如果你换了新地图，比如 my_map.yaml，里面的 image: 也要对应到正确的 .pgm 文件路径，否则 Nav2 会加载失败。



 2. 新测试脚本 mymcp_test/test_arm_sequence.py

# 完整捡球序列（init→prepare_grab→open_gripper→grab→init）
  python3 test_arm_sequence.py

# 只测单个动作
  python3 test_arm_sequence.py --action init
  python3 test_arm_sequence.py --action prepare_grab
  python3 test_arm_sequence.py --action open_gripper
  python3 test_arm_sequence.py --action grab

  python3 test_arm_sequence.py --action init

# 列出所有动作组指令
  python3 test_arm_sequence.py --list

# 调整动作间隔（默认2秒）
  python3 test_arm_sequence.py --interval 3.0

# 指定不同串口
  python3 test_arm_sequence.py --port /dev/ttyUSB1

  ---
  拍摄视频的建议

  核心原则：把"全自动一气呵成"变成"分段可控"。

  准备阶段（拍之前）

  3. 先单独跑 test_arm_sequence.py 把抓取动作调好，确认每步舵机都到位
  4. 固定羽毛球位置：在地上贴胶带标记，每次拍摄都把球放到同一位置，减少30秒内手动调整的量
  5. 确认导航点坐标：让机器人先跑一次空导航，看它停在哪里，再决定羽毛球放哪

  拍摄策略（分段拍+剪辑）

  ┌───────┬────────────────────────┬──────────────────────────────┐
  │ 段落  │          内容          │             技巧             │
  ├───────┼────────────────────────┼──────────────────────────────┤
  │ ①出发 │ 机器人从原点开始导航   │ 广角拍全局，显示地图和轨迹   │
  ├───────┼────────────────────────┼──────────────────────────────┤
  │ ②到达 │ 到达羽毛球区，30秒等待 │ 这段可以剪短，对着机械臂特写 │
  ├───────┼────────────────────────┼──────────────────────────────┤
  │ ③抓取 │ 机械臂执行动作序列     │ 从侧面拍特写，这段是亮点     │
  ├───────┼────────────────────────┼──────────────────────────────┤
  │ ④返回 │ 导航回原点             │ 广角拍，和①对称              │
  └───────┴────────────────────────┴──────────────────────────────┘

  ▎ 30秒等待段：一镜头先拍机器人到位，然后你自然走进去手动微调，不用藏着掖着，这本身就是人机协作的展示。

  应急处理

  - Ctrl+C 中止任务后，立刻 python3 test_arm_sequence.py --action init 让机械臂复位
  - 如果导航跑偏了，直接 keyboard 节点把机器人开回来，不影响重新触发任务
