# Phase 1 详细知识点目录

## C++ 与 C 的本质差异

---

### 第一章：引用（Reference）

**1.1 引用的本质**

- 引用不是变量，是别名——它在底层是什么
- 引用 vs 指针：内存模型的真实差异
- 为什么引用不能为空、不能重新绑定
- 编译器如何实现引用（汇编层面一瞥）

**1.2 引用作为函数参数**

- 传值 / 传指针 / 传引用 的性能与语义差异
- 什么时候用引用传参，什么时候必须用指针
- 嵌入式场景：大结构体传参为什么一定要用引用

**1.3 const 引用**

- `const T&` 能绑定临时对象，为什么指针不行
- 万能的函数入参写法：`const T&` 的哲学
- 常见错误：把 `const T&` 返回值当成安全的（悬垂引用）

**1.4 引用作为返回值**

- 什么场景下函数可以返回引用
- 悬垂引用：最经典的嵌入式内存 Bug 来源
- 链式调用是怎么用引用实现的（`operator<<` 为例）

**1.5 嵌入式中的引用实践**

- HAL 驱动封装中用引用传递配置结构体
- ROS2 回调函数参数为什么几乎全是 `const T&`
- 引用与 `volatile` 的组合：操作寄存器时的写法

### 第二章：const 正确性（Const Correctness）

**2.1 const 的四种位置与含义**

- `const int*`、`int* const`、`const int* const` 彻底分清
- 顶层 const 与底层 const 的概念
- 为什么这个区别在嵌入式里会影响编译结果

**2.2 const 变量与 #define 的本质区别**

- `#define` 是文本替换，`const` 是有类型的常量
- 嵌入式里为什么应该用 `const` 替代 `#define`（调试、类型检查）
- `constexpr` 预告：编译期常量的终极形态

**2.3 函数中的 const**

- `const T&` 入参（第一章已讲，这里深化）
- `const` 成员函数：对象状态的契约
- `mutable` 关键字：const 函数里的"例外"

**2.4 const 与指针在嵌入式中的实战**

- 硬件寄存器地址映射：为什么用 `volatile const`
- Flash 中的常量表：`const` 数组如何保证不被放到 RAM
- 函数指针与 const 的组合写法

**3.3 命名空间在工程中的最佳实践**

- 嵌入式驱动库的命名空间设计规范
- ROS2 节点的典型命名空间结构

**3.4 ADL（实参依赖查找）**

### 第四章：函数重载（Function Overloading）

**4.1 重载的本质：名字修饰**

- 编译器如何区分同名函数（Name Mangling 机制）

**4.2 重载决议规则**

- 精确匹配 > 提升 > 标准转换 > 用户自定义转换
- 二义性错误：什么情况下编译器会拒绝

**4.3 默认参数 vs 重载**

- 两者的使用场景与权衡
- 默认参数与重载同时存在时的二义性
- 嵌入式驱动 API 设计：用哪个更合适

**4.4 重载与 const 的交互**

- `const` 成员函数重载的实际意义
- 同一个 `operator[]` 的 const 和非 const 版本
- 嵌入式容器类的实现示例

**4.5 运算符重载预告**

- 运算符重载本质上也是函数重载
