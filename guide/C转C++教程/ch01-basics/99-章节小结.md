# 章节小结

本章介绍了从 C 语言过渡到 C++ 的基础知识，涵盖 10 个核心知识点。以下是每个小节的核心要点回顾。

---

## 01 — 使用 C++ 刷算法的好处

- C++ 几乎完全兼容 C 语法，存量 C 代码可以直接在 C++ 编译器中编译运行，仅需在类型转换等少数地方做微调。
- STL（标准模板库）提供了 vector、set、map、stack、queue 等开箱即用的数据结构，以及 sort、binary_search、accumulate 等算法库，大幅减少编码量。
- string 类提供了运算符重载（`+`、`==`、`[]` 等）和成员函数（`substr`、`find`、`length` 等），字符串操作比 C 语言的 char[] 便捷得多。
- 通过 `ios::sync_with_stdio(false); cin.tie(0);` 可以将 cin/cout 的性能提升到接近 scanf/printf 的水平，足以应对绝大多数竞赛场景。

---

## 02 — using namespace std

- 命名空间（namespace）是 C++ 解决名称冲突的机制，std 是标准库的命名空间，所有标准库组件都定义在其中。
- `using namespace std;` 可以省去 `std::` 前缀，使代码更简洁，但可能引发命名冲突。
- 替代方案包括：始终使用 `std::` 前缀（最安全）、只引入单个名称（`using std::cout;`）、或在函数内部局部使用。
- 算法竞赛中放心使用 `using namespace std;`，但大型工程中推荐显式使用 `std::` 前缀。

---

## 03 — cin 和 cout

- `cin >> x` 和 `cout << x` 底层是运算符重载，返回值是流对象本身，支持链式调用。
- 默认情况下 cin/cout 与 C 标准 IO 同步以保证混用安全，但这带来了额外的性能开销。
- 在程序开头加上 `ios::sync_with_stdio(false); cin.tie(0);` 可大幅提升 IO 性能，但加速后不能再混用 scanf/printf。
- `endl` 会刷新缓冲区，性能低于 `\n`；除调试场景外优先使用 `\n`。

---

## 04 — 头文件

- C++ 提供了 C++ 风格的头文件，规则是去掉 `.h` 加 `c` 前缀（如 `<stdio.h>` → `<cstdio>`）。
- C++ 特有的头文件包括 `<iostream>`、`<vector>`、`<string>`、`<algorithm>` 等，提供了 STL 容器和算法支持。
- C++ 风格的头文件将名称放在 std 命名空间中，更规范； `<xxx.h>` 将名称放在全局命名空间。
- `<bits/stdc++.h>` 万能头文件在竞赛中可用，但不是标准 C++，正式工程中不应使用。

---

## 05 — 变量声明

- C89 要求在函数或代码块的开头集中声明变量，C++ 允许在任意位置声明变量（就近声明）。
- for 循环内部可以直接声明循环变量（`for (int i = 0; ...)`），其作用域严格限制在循环体内。
- 就近声明缩小了变量的作用域，减少了未初始化变量和变量意外误用的风险。
- C++17 起还支持在 if/switch 语句中声明变量（`if (int x = func(); x > 0) { ... }`），进一步缩小作用域。

---

## 06 — bool 变量

- C++ 原生支持 bool 类型，是独立的基础数据类型，取值为 `true`（1）和 `false`（0），无需任何头文件。
- C 语言中原先没有 bool 类型（C99 才通过 `_Bool` 和 `<stdbool.h>` 提供），通常用 int 的 0/1 模拟。
- 各种类型到 bool 的转换规则：0 值（或空值）转换为 false，任何非 0 值转换为 true。
- 使用 `boolalpha` 可以控制 cout 输出 true/false 字符串而非 0/1。

---

## 07 — const 定义常量

- `const` 定义常量后值不可修改，其作用域遵循 C++ 的作用域规则，由编译器处理，支持类型检查。
- 与 `#define` 相比：const 有类型检查、遵循作用域规则、符号表可见便于调试；#define 是预处理阶段的纯文本替换，没有作用域和类型检查。
- const 可用于修饰变量、指针（顶层/底层 const）、函数参数、函数返回值、成员函数等。
- 在 C++ 中定义常量时应优先使用 `const` 而非 `#define`（C++11 起还可用 `constexpr` 定义编译期常量）。

---

## 08 — string 类

- C++ string 是类类型，自动管理内存，支持动态增长，不会像 C 的 char[] 一样出现缓冲区溢出问题。
- 常用操作：`s.length()` / `s.size()`（O(1) 获取长度）、`s.substr(pos, count)`（截取子串）、`s.find(str)`（查找）、`s.append(str)` / `s + str`（拼接）、`s.replace(pos, len, str)`（替换）。
- string 支持运算符重载：`+` 拼接、`==` / `!=` / `<` / `>` 比较、`[]` 下标访问。
- C++11 起有 `to_string()` 将数值转为字符串，和 `stoi()` / `stod()` 将字符串转为数值。需要与 C 库交互时通过 `c_str()` 获取 `const char*`。

---

## 09 — 结构体

- C++ 结构体声明变量时无需 `struct` 关键字，直接使用类型名即可（`Student s;` 而非 `struct Student s;`）。
- C++ 结构体可以包含成员函数、构造函数、析构函数等面向对象特性，与 class 的唯一定义区别是默认访问权限：struct 默认 public，class 默认 private。
- 结构体可以嵌套、可以用于 STL 容器、可以作为函数参数（大结构体建议传 const 引用）。
- 算法竞赛中常用 struct 定义简单的数据结构（如 Point、Edge、Node），配合初始化列表和成员函数使用。

---

## 10 — 引用和传值

- 传值（值传递）复制一份副本，函数内修改不影响原变量；传引用（引用传递）操作原变量，不复制，可以修改原变量。
- 引用是变量的别名：必须初始化、不能为空、不能重新绑定；指针可以指向不同对象、可以为空、需要手动解引用。
- const 引用（`const T&`）是函数参数的最佳实践：不复制、不修改原变量，还能接收临时对象和字面量。
- 引用常用于：修改函数外部变量、实现多返回值、范围 for 循环中修改元素、避免大对象复制的开销。
- 不要返回局部变量的引用，会导致悬空引用（未定义行为）。
