# 头文件

## 一、C 头文件 vs C++ 头文件

C++ 完全兼容 C 的标准库，但提供了**C++ 风格**的头文件（去掉 `.h`，加 `c` 前缀）。

### 1.1 完整对照表

| C 头文件 | C++ 风格 | 主要功能 |
|---------|---------|---------|
| `<assert.h>` | `<cassert>` | 断言宏 assert |
| `<ctype.h>` | `<cctype>` | 字符处理（isalpha、isdigit、toupper 等） |
| `<errno.h>` | `<cerrno>` | 错误码 errno |
| `<float.h>` | `<cfloat>` | 浮点数限制常量 |
| `<limits.h>` | `<climits>` | 整数类型大小限制（INT_MAX 等） |
| `<locale.h>` | `<clocale>` | 本地化 |
| `<math.h>` | `<cmath>` | 数学函数（sqrt、sin、cos 等） |
| `<setjmp.h>` | `<csetjmp>` | 非局部跳转 setjmp/longjmp |
| `<signal.h>` | `<csignal>` | 信号处理 |
| `<stdarg.h>` | `<cstdarg>` | 可变参数列表 |
| `<stddef.h>` | `<cstddef>` | 常用类型和宏（size_t、NULL） |
| `<stdio.h>` | `<cstdio>` | 标准 IO（printf、scanf 等） |
| `<stdlib.h>` | `<cstdlib>` | 通用工具（malloc、atoi、rand 等） |
| `<string.h>` | `<cstring>` | 字符串处理（strlen、strcpy 等） |
| `<time.h>` | `<ctime>` | 时间处理（time、clock 等） |

### 1.2 C++ 特有的头文件

C++ 新增了大量标准库头文件，以下是算法竞赛中最常用的：

| 头文件 | 主要内容 | 使用频率 |
|-------|---------|---------|
| `<iostream>` | cin、cout、cerr、endl | 必用 |
| `<iomanip>` | setw、setprecision 等 IO 控制 | 常用 |
| `<string>` | string 类、getline、to_string | 必用 |
| `<vector>` | vector 动态数组 | 必用 |
| `<algorithm>` | sort、find、binary_search、max、min 等 | 必用 |
| `<numeric>` | accumulate、iota、partial_sum | 常用 |
| `<map>` | map（有序映射） | 常用 |
| `<unordered_map>` | unordered_map（哈希映射，C++11） | 常用 |
| `<set>` | set（有序集合） | 常用 |
| `<unordered_set>` | unordered_set（哈希集合，C++11） | 常用 |
| `<stack>` | stack（栈） | 常用 |
| `<queue>` | queue（队列）、priority_queue（优先队列） | 常用 |
| `<deque>` | deque（双端队列） | 常用 |
| `<list>` | list（双向链表） | 偶尔 |
| `<utility>` | pair、make_pair | 常用 |
| `<tuple>` | tuple（元组，C++11） | 偶尔 |
| `<functional>` | less、greater、hash、function | 常用 |
| `<iterator>` | 迭代器相关（begin、end） | 常用 |
| `<memory>` | shared_ptr、unique_ptr、make_shared | 偶尔 |

| `<fstream>` | 文件流（ifstream、ofstream） | 偶尔 |

| `<type_traits>` | 类型检查（is_same、is_pointer 等） | 极少 |

### 1.3 转换规则

从 C 头文件到 C++ 风格头文件的规则非常简单：

> **去掉 `.h` 后缀，在开头加字母 `c`**

```
<string.h>  →  <cstring>
<math.h>    →  <cmath>
<stdio.h>   →  <cstdio>
<stdlib.h>  →  <cstdlib>
```

## 二、为什么 C++ 要去掉 .h 加 c 前缀

### 2.1 历史原因

C++ 诞生时，为了兼容 C 的标准库，直接沿用了 `<stdio.h>` 等头文件。但 C++ 标准委员会希望区分两个点：

1. **标识头文件是 C++ 版本**：`<cstdio>` 中的函数被声明在 `std` 命名空间中，而 `<stdio.h>` 中的函数在全局命名空间
2. **避免文件名冲突**：某些操作系统不区分大小写，`.h` 文件容易混淆

### 2.2 细微差异

实际上，`<cstdio>` 和 `<stdio.h>` 并不完全等价：

```cpp
#include <cstdio>
// printf 在 std 命名空间中
std::printf("Hello\n");

// 某些实现也将其放在全局命名空间，但标准不保证
```

```cpp
#include <stdio.h>
// printf 在全局命名空间中
printf("Hello\n");
```

在 C++ 中，`<cxxx>` 形式的头文件将名称放在 `std` 命名空间中，也可能放在全局命名空间。而 `<xxx.h>` 将名称放在全局命名空间，也可能放在 `std` 命名空间。

**实际建议**：在 C++ 中优先使用 `<cxxx>` 形式的头文件，配合 `using namespace std;` 一起使用。

## 三、完整可运行代码：多种头文件的实际应用

```cpp
#include <iostream>      // cin, cout
#include <string>        // string
#include <vector>        // vector
#include <algorithm>     // sort, max, min
#include <numeric>       // accumulate
#include <cmath>         // sqrt, pow
#include <iomanip>       // setprecision, fixed
#include <cstdlib>       // rand, srand (C 风格)
#include <ctime>         // time (C 风格)
#include <climits>       // INT_MAX, INT_MIN
#include <map>           // map
#include <set>           // set
using namespace std;

int main() {
    // ============ 1. iostream ============
    cout << "=== C++ 头文件综合示例 ===\n\n";

    // ============ 2. string ============
    string greeting = "Hello, C++ World!";
    cout << "string: " << greeting << "\n";
    cout << "长度: " << greeting.length() << "\n\n";

    // ============ 3. vector + algorithm + numeric ============
    vector<int> nums = {5, 2, 8, 1, 9, 3, 7, 4, 6};

    cout << "原始数组: ";
    for (int x : nums) cout << x << " ";
    cout << "\n";

    sort(nums.begin(), nums.end());
    cout << "排序后:   ";
    for (int x : nums) cout << x << " ";
    cout << "\n";

    int sum = accumulate(nums.begin(), nums.end(), 0);
    int max_val = *max_element(nums.begin(), nums.end());
    int min_val = *min_element(nums.begin(), nums.end());
    int avg = sum / (int)nums.size();

    cout << "sum = " << sum << "\n";
    cout << "max = " << max_val << "\n";
    cout << "min = " << min_val << "\n";
    cout << "avg = " << avg << "\n\n";

    // ============ 4. cmath ============
    double x = 9.0;
    cout << "sqrt(" << x << ") = " << sqrt(x) << "\n";
    cout << "pow(2, 10) = " << pow(2, 10) << "\n\n";

    // ============ 5. iomanip ============
    double pi = 3.14159265358979;
    cout << "默认输出: " << pi << "\n";
    cout << fixed << setprecision(4);
    cout << "保留4位小数: " << pi << "\n";
    cout << setprecision(10);
    cout << "保留10位小数: " << pi << "\n\n";

    // ============ 6. cstdlib + ctime ============
    srand(time(0));
    cout << "随机数 (1~100): " << (rand() % 100 + 1) << "\n\n";

    // ============ 7. climits ============
    cout << "int 范围: " << INT_MIN << " ~ " << INT_MAX << "\n\n";

    // ============ 8. map ============
    map<string, int> scores;
    scores["Alice"] = 95;
    scores["Bob"] = 87;
    scores["Charlie"] = 92;

    cout << "map 遍历:\n";
    for (auto& [name, score] : scores) {
        cout << "  " << name << ": " << score << "\n";
    }
    cout << "\n";

    // ============ 9. set ============
    set<int> unique_nums = {3, 1, 4, 1, 5, 9, 2, 6, 5, 3};
    cout << "set (自动去重+排序): ";
    for (int n : unique_nums) cout << n << " ";
    cout << "\n";

    return 0;
}
```

**运行结果**：
```
=== C++ 头文件综合示例 ===

string: Hello, C++ World!
长度: 19

原始数组: 5 2 8 1 9 3 7 4 6 
排序后:   1 2 3 4 5 6 7 8 9 
sum = 45
max = 9
min = 1
avg = 5

sqrt(9) = 3
pow(2, 10) = 1024

默认输出: 3.14159
保留4位小数: 3.1416
保留10位小数: 3.1415926536

随机数 (1~100): 73

int 范围: -2147483648 ~ 2147483647

map 遍历:
  Alice: 95
  Bob: 87
  Charlie: 92

set (自动去重+排序): 1 2 3 4 5 6 9 
```

## 四、万能头文件

在某些在线评测系统（如 Codeforces、AtCoder）中，可以使用**万能头文件**：

```cpp
#include <bits/stdc++.h>
```

这个头文件包含了 C++ 几乎所有的标准库头文件，一劳永逸。

### 优点

- 再也不用记具体的头文件了
- 编译速度在竞赛环境中可以接受

### 缺点

- **不是标准 C++**， VS 编译器可能没有这个文件
- 编译速度慢（包含大量不需要的头文件）
- 会导致代码的依赖不明确
- 正式工程中绝对不要使用

### 使用建议

```cpp
// 刷算法题时可以这样写
#include <bits/stdc++.h>
using namespace std;

int main() {
    // 直接使用各种 STL，无需额外包含头文件
    vector<int> v;
    string s;
    map<int, int> m;
    // ...
    return 0;
}
```

但在正式学习和工程中，建议逐个包含需要的头文件。

## 五、头文件包含的注意事项和常见坑

### 5.1 重复包含

同一个头文件被多次包含可能导致重复定义错误。使用**头文件守卫**（include guard）：

```cpp
// myheader.h
#ifndef MYHEADER_H     // 如果没有定义 MYHEADER_H
#define MYHEADER_H     // 则定义它

void myFunction();

#endif                 // 结束守卫
```

或者使用 `#pragma once`（更简洁，但非标准）：

```cpp
// myheader.h
#pragma once

void myFunction();
```

### 5.2 循环包含

头文件 A 包含头文件 B，头文件 B 又包含头文件 A，导致编译死循环或奇怪的错误。解决方案：使用**前向声明**（forward declaration）。

### 5.3 头文件不要使用 using namespace std

```cpp
// WRONG.h — 错误示例
#include <string>
using namespace std;

string greet();  // 会污染所有包含此头文件的文件
```

### 5.4 头文件顺序

虽然通常不影响编译，但建议按照以下顺序包含：
1. 自定义头文件
2. 第三方库头文件
3. C++ 标准库头文件
4. C 标准库头文件

## 六、总结

| 要点 | 说明 |
|------|------|
| C 头文件转 C++ | 去掉 `.h` 加 `c` 前缀（`<stdio.h>` → `<cstdio>`） |
| C++ 特有头文件 | `<iostream>`、`<vector>`、`<string>`、`<algorithm>` 等 |
| C++ 风格优点 | 名称在 `std` 命名空间中，更规范 |
| 万能头文件 | `<bits/stdc++.h>` — 竞赛可用，工程勿用 |
| 最佳实践 | 需要什么包含什么，学会使用 `#include` 守卫 |
