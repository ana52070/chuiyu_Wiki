# 变量声明

## 一、C 语言：必须在函数开头声明

在 C89/C90 标准中，所有局部变量**必须在函数或代码块的开头声明**，不能与语句混合。

```c
// C89 风格 — 变量必须在开头集中声明
#include <stdio.h>

int main() {
    int i;           // 变量声明必须在开头
    int sum = 0;
    int arr[10];

    for (i = 0; i < 10; i++) {
        arr[i] = i;
    }

    // 错误：不能在中间声明变量（C89）
    // int x = 10;   // C89 不允许

    for (i = 0; i < 10; i++) {
        sum += arr[i];
    }

    printf("sum = %d\n", sum);
    return 0;
}
```

C99 标准放宽了这一限制，允许在代码块任意位置声明变量。但很多教学和竞赛环境仍以 C89 为主流。

## 二、C++：随时声明，就近使用

C++ 允许在任意位置声明变量，这带来两个好处：

1. **变量在使用处声明，提高可读性**
2. **缩小变量的作用域，减少错误**

```cpp
#include <iostream>
#include <vector>
#include <string>
using namespace std;

int main() {
    // 在需要的地方声明
    cout << "请输入人数: ";
    int n;                // 在第一次使用处声明
    cin >> n;

    vector<string> names;
    for (int i = 0; i < n; i++) {
        cout << "请输入第 " << i + 1 << " 个人的名字: ";
        string name;      // 循环内部声明，每次循环都重新创建
        cin >> name;
        names.push_back(name);
    }

    // 输出
    cout << "\n名单:\n";
    for (int i = 0; i < n; i++) {
        cout << i + 1 << ". " << names[i] << endl;
    }

    return 0;
}
```

**运行结果示例**：
```
请输入人数: 3
请输入第 1 个人的名字: Alice
请输入第 2 个人的名字: Bob
请输入第 3 个人的名字: Charlie

名单:
1. Alice
2. Bob
3. Charlie
```

## 三、for 循环内声明变量

C++ 最重要的特性之一：在 `for` 循环的初始化语句中声明变量。

```cpp
#include <iostream>
using namespace std;

int main() {
    // C++ 风格：在 for 循环内声明
    for (int i = 0; i < 5; i++) {
        cout << i << " ";
    }
    cout << endl;

    // 这里 i 已经不可访问
    // cout << i;  // 编译错误：i 未定义

    // 嵌套循环互不干扰
    for (int i = 0; i < 3; i++) {
        for (int j = 0; j < 3; j++) {
            cout << "(" << i << "," << j << ") ";
        }
        cout << endl;
    }

    return 0;
}
```

**运行结果**：
```
0 1 2 3 4 
(0,0) (0,1) (0,2) 
(1,0) (1,1) (1,2) 
(2,0) (2,1) (2,2) 
```

### 作用域细节

```cpp
for (int i = 0; i < 10; i++) {
    // i 的作用域从声明处到循环结束
}
// i 在这里已经销毁

// 可以在不同的循环中重复使用相同的变量名
for (int i = 0; i < 5; i++) {  // 新的 i
    cout << i;
}
```

如果需要循环结束后仍能访问 `i`，则需要将变量声明在外部：

```cpp
int i;
for (i = 0; i < 10; i++) {
    // ...
}
// i 在这里仍然可用
cout << "循环结束时的 i = " << i << endl;  // i = 10
```

## 四、完整对比：C 风格 vs C++ 风格

```cpp
#include <iostream>
#include <vector>
#include <string>
using namespace std;

int main() {
    // ============ 场景 1：一次性使用的临时变量 ============
    cout << "=== 场景1: 临时变量 ===\n";

    // C 风格：提前声明，可能忘记初始化
    int a;              // 声明但不定义
    // ... 中间可能有很多行代码 ...
    a = 10;             // 使用

    // C++ 风格：在使用处声明并初始化
    int b = 10;         // 清晰明确

    // ============ 场景 2：循环变量 ============
    cout << "\n=== 场景2: 循环变量 ===\n";

    // C 风格：变量声明在函数开头或循环外部
    int j;
    for (j = 0; j < 5; j++) {
        cout << j << " ";
    }
    cout << "\n循环结束后 j = " << j << "\n";  // j = 5

    // C++ 风格：在 for 内声明
    for (int i = 0; i < 5; i++) {
        cout << i << " ";
    }
    cout << "\n";
    // 这里 i 已经不可访问

    // ============ 场景 3：条件分支中声明变量 ============
    cout << "\n=== 场景3: 条件分支 ===\n";

    // C++17 起，可以在 if/switch 中声明变量
    // 这在 C 中根本不支持
    if (int x = 10; x > 5) {
        cout << "x = " << x << " 大于 5\n";
    }
    // x 在这里已经不可访问

    // ============ 场景 4：动态大小的容器 ============
    cout << "\n=== 场景4: 动态容器 ===\n";

    // C 语言需要固定大小或动态分配
    // int fixed[100];             // 固定大小
    // int n = 10;
    // int* dyn = malloc(n * sizeof(int));  // 动态分配

    // C++ 直接用 vector，且声明在需要的地方
    vector<int> v = {3, 1, 4, 1, 5};
    v.push_back(9);
    for (int x : v) {
        cout << x << " ";
    }
    cout << "\n";

    return 0;
}
```

**运行结果**：
```
=== 场景1: 临时变量 ===

=== 场景2: 循环变量 ===
0 1 2 3 4 
循环结束后 j = 5
0 1 2 3 4 

=== 场景3: 条件分支 ===
x = 10 大于 5

=== 场景4: 动态容器 ===
3 1 4 1 5 9 
```

## 五、注意事项和常见坑

### 5.1 未初始化变量的风险

C++ 中局部变量不会自动初始化（除非是特定类型如 `std::string`）：

```cpp
int x;           // 未初始化，值是不确定的（可能是垃圾值）
cout << x;       // 危险！可能会输出任意值

string s;        // string 会自动初始化为空字符串
cout << s;       // 安全，输出空字符串
```

**规则**：
- 基本类型（int、double、char 等）的局部变量不会自动初始化
- 类类型（string、vector 等）会通过默认构造函数自动初始化
- 全局变量和静态变量会初始化为 0

### 5.2 变量隐藏（shadowing）

内层作用域的变量会隐藏外层同名变量：

```cpp
#include <iostream>
using namespace std;

int main() {
    int x = 10;              // 外层的 x

    {
        int x = 20;          // 内层的 x，隐藏了外层的 x
        cout << "内层 x = " << x << endl;  // 20
        x = 30;              // 修改的是内层的 x
    }

    cout << "外层 x = " << x << endl;      // 10，外层的 x 不受影响

    return 0;
}
```

**运行结果**：
```
内层 x = 20
外层 x = 10
```

### 5.3 for 循环变量作用域的常见误解

```cpp
// C++98/03 中，for 循环中声明的变量在循环外也可访问（编译器 bug）
// 但 C++11 及以后标准中，循环变量作用域严格限制在循环内

for (int i = 0; i < 10; i++) {
    // ...
}
// i 在 C++11 后不再可访问
```

### 5.4 结构化绑定（C++17）

```cpp
#include <iostream>
#include <map>
using namespace std;

int main() {
    map<string, int> scores;
    scores["Alice"] = 95;

    // C++17 结构化绑定：在 if 中声明多个变量
    if (auto [iter, inserted] = scores.insert({"Bob", 87}); inserted) {
        cout << "插入成功: " << iter->first << " -> " << iter->second << endl;
    }

    // 在 for 中使用结构化绑定
    for (auto& [name, score] : scores) {
        cout << name << ": " << score << endl;
    }

    return 0;
}
```

## 六、总结

| 特性 | C 语言 | C++ |
|------|--------|-----|
| 变量声明位置 | C89 必须在开头 | 任意位置 |
| for 循环变量 | 必须在外部声明 | 可在 for 内声明 |
| 变量作用域 | 较大（声明到块结束） | 更精细（接近使用处） |
| 初始化要求 | 必须手动初始化 | 同 C，但类类型自动初始化 |
| 安全性 | 较低（可能误用未初始化变量） | 较高（就近声明减少错误） |

**核心建议**：
1. 尽量在变量第一次使用时声明
2. 在 for 循环内部声明循环变量
3. 声明时尽量同时初始化
4. 避免变量隐藏（同名变量在不同作用域）
