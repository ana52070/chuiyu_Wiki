## 章节小结：STL 容器速查

本章介绍了 C++ STL 中 6 种最常用的容器，涵盖**序列容器**、**有序关联容器**、**无序关联容器**和**容器适配器**四大类。

---

### 一、容器概览与头文件

| 容器 | 头文件 | 类型 | 数据结构 | 元素特性 | 适用场景 |
|------|--------|------|---------|---------|---------|
| `vector` | `<vector>` | 序列容器 | 动态数组 | 连续内存，自动扩容 | 随机访问频繁，尾部增删 |
| `set` | `<set>` | 有序关联容器 | 红黑树 | 唯一、有序 | 去重、范围查询、有序遍历 |
| `map` | `<map>` | 有序关联容器 | 红黑树 | 键值对、键唯一、有序 | 字典、映射、键值查找 |
| `stack` | `<stack>` | 容器适配器 | deque 适配 | FILO（先进后出） | 括号匹配、DFS、Undo |
| `queue` | `<queue>` | 容器适配器 | deque 适配 | FIFO（先进先出） | BFS、任务调度、缓冲区 |
| `unordered_map` | `<unordered_map>` | 无序关联容器 | 哈希表 | 键值对、键唯一、无序 | 快速查找，不需要顺序 |
| `unordered_set` | `<unordered_set>` | 无序关联容器 | 哈希表 | 唯一、无序 | 快速去重判断 |

### 二、时间复杂度速查表

| 操作 | `vector` | `set` | `map` | `stack` | `queue` | `unordered_map` / `unordered_set` |
|------|---------|-------|-------|---------|---------|----------------------------------|
| 插入/入栈/入队 | 尾部 **均摊 O(1)**，中间 O(n) | **O(log n)** | **O(log n)** | **O(1)** | **O(1)** | **均摊 O(1)** |
| 查找 | O(n)（未排序） | **O(log n)** | **O(log n)** | — | — | **均摊 O(1)** |
| 删除 | 尾部 O(1)，中间 O(n) | **O(log n)** | **O(log n)** | **O(1)** | **O(1)** | **均摊 O(1)** |
| 随机访问 | **O(1)** | — | — | — | — | — |
| 范围查询 | 排序后 O(log n) | **O(log n)** | **O(log n)** | — | — | **不支持** |

### 三、容器核心要点

#### vector（动态数组）

```
头文件: <vector>
创建:  vector<int> v;          // 空
       vector<int> v(10);      // 10个0
       vector<int> v(10, 2);   // 10个2
       vector<int> v = {1,2,3};// 初始化列表
遍历:  for (int x : v)  |  for (size_t i = 0; i < v.size(); i++)
重点:  push_back/pop_back, size/capacity, front/back, []/at
性能:  随机访问 O(1)，尾部增删均摊 O(1)，中间插入 O(n)
坑:    [] 不检查越界，at() 检查越界；扩容时迭代器失效
```

#### set（有序集合）

```
头文件: <set>
创建:  set<int> s;             // 空
       set<int> s = {3,1,4};   // 初始化列表，自动去重排序
遍历:  for (int x : s)         // 升序
重点:  insert/erase/find/count, lower_bound/upper_bound
性能:  所有操作 O(log n)
坑:    set 元素不可直接修改；重复插入被忽略
变体:  multiset（允许重复元素）
```

#### map（有序键值对）

```
头文件: <map>
创建:  map<string, int> m;
       map<string, int> m = {{"a",1}, {"b",2}};
遍历:  for (const auto& [k, v] : m)     // C++17 结构化绑定
       for (auto it = m.begin(); ...)   // it->first / it->second
重点:  []/at/insert/find/erase, insert_or_assign (C++17)
性能:  所有操作 O(log n)
坑:    [] 访问不存在的键会默认创建！用 at() 或 find() 安全读取
变体:  multimap（允许重复键）
```

#### stack（栈）

```
头文件: <stack>
创建:  stack<int> s;
操作:  push(), pop(), top(), size(), empty()
性能:  所有操作 O(1)
坑:    空栈调用 top()/pop() = 未定义行为（崩溃！）
       pop() 不返回值，要先 top() 再 pop()
       stack 没有迭代器，不支持遍历
应用:  括号匹配、表达式求值、浏览器后退
```

#### queue（队列）

```
头文件: <queue>
创建:  queue<int> q;
操作:  push(), pop(), front(), back(), size(), empty()
性能:  所有操作 O(1)
坑:    空队列调用 front/back/pop() = 未定义行为
       pop() 不返回值，要先 front() 再 pop()
       queue 没有迭代器，不支持遍历
变体:  priority_queue（优先队列，默认大顶堆，push/pop O(log n)）
应用:  BFS、任务队列、缓冲处理
```

#### unordered_map / unordered_set（无序哈希容器）

```
头文件: <unordered_map> / <unordered_set>
创建:  unordered_map<string, int> um;
       unordered_set<int> us = {3,1,4};
遍历:  for (const auto& [k, v] : um)   // 顺序不确定！
       for (int x : us)                 // 顺序不确定！
接口:  与 map/set 基本一致，但不支持 lower_bound/upper_bound
性能:  均摊 O(1)，最坏 O(n)
坑:    不要依赖任何特定顺序；rehash 时会重新排列所有元素
建议:  追求性能时使用，需要顺序时用 map/set
优化:  reserve(n) 预分配空间避免 rehash
```

### 四、选择决策树

```
需要什么样的数据结构？
│
├─ 需要随机访问（下标）？ ───────────────▶ vector
│
├─ 需要键值对？
│   ├─ 需要按键排序？ ──────────────────▶ map
│   └─ 不需要排序，追求速度？ ─────────▶ unordered_map
│
├─ 只需要判断存在/去重？
│   ├─ 需要有序？ ──────────────────────▶ set
│   └─ 不需要有序，追求速度？ ─────────▶ unordered_set
│
├─ 需要 FILO（先进后出）？ ────────────▶ stack
│
└─ 需要 FIFO（先进先出）？
    ├─ 普通 FIFO？ ─────────────────────▶ queue
    └─ 需要优先级调度？ ────────────────▶ priority_queue
```

### 五、代码模板速览

```cpp
// ===== vector =====
#include <vector>
vector<int> v = {3, 1, 4};
v.push_back(1); v.pop_back();
v[0]; v.at(0); v.front(); v.back();
v.size(); v.empty(); v.resize(10);

// ===== set =====
#include <set>
set<int> s = {3, 1, 4, 1};          // s = {1, 3, 4}
s.insert(5); s.erase(3);
s.find(4); s.count(1);
s.lower_bound(2); s.upper_bound(4);

// ===== map =====
#include <map>
map<string, int> m = {{"a",1}, {"b",2}};
m["c"] = 3; m.insert({"d", 4});
m.at("a"); m.find("b"); m.count("c");
m.erase("d");

// ===== stack =====
#include <stack>
stack<int> st;
st.push(1); st.push(2);
st.top(); st.pop();
st.size(); st.empty();

// ===== queue =====
#include <queue>
queue<int> q;
q.push(1); q.push(2);
q.front(); q.back(); q.pop();
q.size(); q.empty();

// ===== unordered_map / unordered_set =====
#include <unordered_map>
#include <unordered_set>
unordered_map<string, int> um = {{"a",1}, {"b",2}};
unordered_set<int> us = {3, 1, 4, 1};
um["c"] = 3; um.find("a"); um.count("b");
us.insert(5); us.find(3); us.erase(4);
```

### 六、需要进一步了解的方向

- **`deque`**（双端队列）：两端插入删除都是 O(1)，是 stack 和 queue 的默认底层容器
- **`list`** 和 **`forward_list`**：链表容器，插入删除 O(1)，但随机访问 O(n)
- **`multiset` / `multimap`**：允许重复元素的 set/map
- **`emplace` 系列函数**：`emplace()`、`emplace_back()`、`emplace_front()`，直接在容器内构造元素，减少拷贝
- **迭代器失效规则**：不同容器的不同操作会导致迭代器失效，是 C++ STL 的重要进阶话题
