# 第 4 章章节小结 —— C++11 新特性

本章共 6 个小节，系统学习了 C++11 中最常用、最实用的入门特性。以下是每个小节的核心要点回顾。

---

## 1. C++11 简介

- C++11 是 **C++98 之后的第一个重大标准**，是 **"Modern C++"（现代 C++ 的开端）**
- 本章重点学习 6 个特性：`auto`、范围 for、`to_string`、`stoi`/`stod`、`unordered_map`/`unordered_set`、Dev C++ 的 C++11 配置
- 其他重要的 C++11 特性（扩展阅读）：`nullptr`、智能指针、Lambda 表达式、移动语义、初始化列表、`constexpr`、`static_assert`、委托构造函数、`override`/`final`、线程支持库
- 编译时通过 `-std=c++11` 参数开启，可通过 `__cplusplus >= 201103L` 宏检测当前标准

## 2. auto 声明

- **原理**：编译期类型推导，不影响运行时性能，编译后与手写类型完全相同
- **最常用场景**：简化迭代器类型声明，尤其是嵌套模板容器（如 `map<string, vector<int>>::iterator` 简化为 `auto`）
- **配合范围 for**：`auto`、`auto &`、`const auto &` 三种形式灵活使用
- **必须初始化**：`auto x;` 不合法，编译器无法推导类型
- **auto 会忽略引用和 const**：需要时须显式写出 `auto &` 或 `const auto &`

| 写法 | 含义 |
|------|------|
| `auto i = expr` | 复制，忽略 const 和引用 |
| `auto &i = expr` | 引用，可修改原值 |
| `const auto &i = expr` | const 引用，只读且避免复制 |

## 3. 基于范围的 for 循环

- **原理**：语法糖，编译期展开为迭代器循环，无运行时开销
- **三种遍历方式**：

| 方式 | 能否修改 | 是否复制 | 推荐场景 |
|:----:|:-------:|:--------:|---------|
| `for (auto i : c)` | 否 | 是 | 基本类型，不需修改 |
| `for (auto &i : c)` | 是 | 否 | 需要修改元素 |
| `for (const auto &i : c)` | 否 | 否 | 只读遍历复杂类型 |

- **支持**：所有有 `begin()`/`end()` 的容器 + 原生数组
- **重要限制**：遍历过程中**不能增删**容器元素

## 4. to_string

- 将数值**安全、简洁**地转换为 `std::string`
- 支持 `int`、`long`、`long long`、`unsigned` 系列、`float`、`double`、`long double`
- 浮点数**固定保留 6 位小数**，需要自定义格式时用 `ostringstream` 或 `sprintf`
- 使用 `printf` 输出 `string` 时需加 `.c_str()`
- 相比 `sprintf`：无需手动管理缓冲区，类型安全，接口简单

## 5. stoi、stod 等字符串转换函数

- 8 个函数：`stoi`、`stol`、`stoll`、`stoul`、`stoull`、`stof`、`stod`、`stold`
- **第二个参数 `idx`**：获取转换停止位置，用于解析含尾随字符的混合字符串
- **第三个参数 `base`**：指定进制（2~36），`0` 表示自动检测（`0x` → 十六进制，`0` → 八进制）
- **异常处理**：`invalid_argument`（无法转换）和 `out_of_range`（数值越界）
- 相比 C 的 `atoi`：能区分"转换错误"和"值为 0"，接口更安全友好
- 在 C++ 项目中，**优先使用 `stoi` 系列函数**

## 6. Dev C++ 使用 C++11

- Dev C++ **默认使用 C++98 标准**，需手动启用 C++11
- 启用步骤：`Tools` → `Compiler Options` → `Settings` → `Code Generation` → `Language standard` → 选择 `ISO C++11`
- 备选：在 Compiler Options 中添加编译参数 `-std=c++11`
- 验证：使用完整的检测程序，通过 `__cplusplus` 宏或编译 C++11 特性代码
- 版本过旧时建议升级 Dev C++ 或换用其他 IDE

## 附录：C++11 开启方式速查

| 编译器/IDE | 开启方式 |
|:----------:|---------|
| GCC / G++ | `g++ -std=c++11 source.cpp` |
| Dev C++ | Tools → Compiler Options → Settings → Code Generation → Language standard → ISO C++11 |
| Code::Blocks | Settings → Compiler → Compiler flags → `Have g++ follow the C++11 ISO C++11 standard` |
| VS Code (MinGW) | `tasks.json` 的 args 中添加 `"-std=c++11"` |
| Visual Studio | 默认支持（VS 2013+），或项目属性 → C/C++ → Language → C++ Language Standard |

## 附录：C++11 特性速查表

| 特性 | 头文件 | 核心语法 |
|:----:|:------:|---------|
| `auto` | 内置关键字 | `auto x = 42;` |
| 范围 for | 内置语法 | `for (auto &x : container) { ... }` |
| `to_string` | `<string>` | `string s = to_string(3.14);` |
| `stoi` / `stod` 等 | `<string>` | `int a = stoi("123", nullptr, 10);` |
| `unordered_map` | `<unordered_map>` | `unordered_map<K,V> um;` |
| `unordered_set` | `<unordered_set>` | `unordered_set<T> us;` |
