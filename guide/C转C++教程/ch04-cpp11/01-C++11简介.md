# C++11 简介

## 1. 从 C++98 到 C++11：漫长的等待

C++ 的第一个国际标准是 **C++98**（1998 年发布，2003 年修订为 C++03）。在此之后的十多年里，C++ 标准一直没有重大更新。直到 **2011 年**，C++11 标准终于正式发布，这是 C++ 发展史上里程碑式的事件。

C++11 被广泛认为是 **"Modern C++"（现代 C++ 的开端）**。它引入了大量新特性，让 C++ 变得更安全、更高效、更易用，编程风格也发生了根本性的转变。许多之前需要手动实现或借助第三方库的功能，现在直接内置在语言和标准库中。

> 如果说 C++98 奠定了 C++ 的基础，那么 C++11 则彻底重塑了 C++ 的面貌。

## 2. 本章涵盖的 C++11 特性

本章将重点学习以下 6 个 C++11 特性，它们是日常编程中最常用、最实用的改进：

| 特性 | 说明 | 核心价值 |
|------|------|---------|
| `auto` 类型推导 | 编译器根据初始值自动推断变量类型 | 简化代码，尤其是模板和迭代器类型 |
| 基于范围的 for 循环 | 直接遍历容器元素，无需使用迭代器 | 代码更简洁、更安全 |
| `to_string()` | 将数值转换为 string 类型 | 告别 sprintf 和 stringstream |
| `stoi`、`stod` 等转换函数 | 将 string 转换为各种数值类型 | 安全、灵活的字符串转数值 |
| `unordered_map` / `unordered_set` | 基于哈希表的无序关联容器 | 平均 O(1) 的查找性能 |
| Dev C++ 启用 C++11 配置 | IDE 中的编译标准设置 | 让开发环境支持 C++11 |

## 3. 其他重要的 C++11 特性（扩展阅读）

除了本章重点学习的特性之外，C++11 还引入了许多其他强大的特性。这里简要列出，供你后续深入学习：

### 3.1 空指针常量 `nullptr`

在 C++11 之前，空指针用 `NULL` 或 `0` 表示，这经常导致重载解析问题：

```cpp
void f(int);
void f(char*);

f(NULL);    // 调用的是 f(int)，而非 f(char*)！
```

C++11 引入了 `nullptr`，它是一个真正的空指针类型，不会与整数混淆：

```cpp
f(nullptr); // 正确地调用 f(char*)
```

### 3.2 智能指针（Smart Pointers）

`<memory>` 头文件新增了 `shared_ptr`、`unique_ptr` 和 `weak_ptr`，实现了自动内存管理，大大减少了内存泄漏的风险：

```cpp
#include <memory>
using namespace std;

unique_ptr<int> p = make_unique<int>(42);       // C++14
shared_ptr<int> sp = make_shared<int>(100);     // 引用计数自动管理
```

### 3.3 Lambda 表达式（匿名函数）

Lambda 表达式允许在函数内部定义匿名函数对象，极大地增强了 STL 算法的可用性：

```cpp
vector<int> v = {5, 2, 8, 1, 9};
sort(v.begin(), v.end(), [](int a, int b) { return a > b; }); // 降序排列
```

### 3.4 移动语义（Move Semantics）

通过右值引用 `&&` 实现"资源窃取"，避免不必要的深拷贝，大幅提升性能。`std::move()` 将左值转换为右值引用。

### 3.5 初始化列表（Initializer List）

统一的初始化语法 `{}`，以及 `std::initializer_list` 支持：

```cpp
vector<int> v = {1, 2, 3, 4, 5};   // C++11 之前不行
map<string, int> m = {{"a", 1}, {"b", 2}};
```

### 3.6 其他亮点

- **`constexpr`**：编译期常量表达式
- **`static_assert`**：编译期断言
- **委托构造函数**：一个构造函数调用另一个构造函数
- **`default` / `delete`**：控制默认函数的生成
- **`override` / `final`**：更安全的虚函数重写控制
- **线程支持库**：`<thread>`、`<mutex>`、`<future>` 等

## 4. 如何开启 C++11 编译

要让编译器使用 C++11 标准，需要在编译时添加特定的命令行选项。

### 4.1 GCC / G++（MinGW）

```bash
g++ -std=c++11 source.cpp -o program
```

如果编译器版本较新（GCC 6+），也可以使用 `-std=c++14` 或 `-std=c++17`，它们都向后兼容 C++11。

### 4.2 MSVC（Visual Studio）

Visual Studio 2013 及以上版本默认就支持 C++11，无需特殊设置。对于 VS 2010/2012，需要在项目属性中设置。

### 4.3 在 IDE 中设置

- **Dev C++**：Tools → Compiler Options → Settings → Code Generation → Language standard → ISO C++11
- **Code::Blocks**：Settings → Compiler → Compiler settings → Compiler flags → 勾选 `Have g++ follow the C++11 ISO standard`
- **Visual Studio**：项目属性 → C/C++ → Language → C++ Language Standard → 选择 C++11 或更高版本

## 5. 检验是否支持 C++11

以下程序使用了多个 C++11 特性，如果它能正常编译运行，说明你的环境已支持 C++11：

```cpp
#include <iostream>
#include <vector>
#include <string>
#include <map>
using namespace std;

int main() {
    // auto 类型推导（C++11）
    auto x = 42;
    cout << "auto 推导出类型: " << x << endl;

    // 初始化列表（C++11）
    vector<int> v = {10, 20, 30, 40, 50};

    // 基于范围的 for 循环 + auto（C++11）
    cout << "范围 for 循环遍历: ";
    for (auto i : v) {
        cout << i << " ";
    }
    cout << endl;

    // to_string（C++11）
    string s = to_string(3.14159);
    cout << "to_string 结果: " << s << endl;

    // stod（C++11）
    double d = stod("2.71828");
    cout << "stod 结果: " << d << endl;

    // 检测 __cplusplus 宏
    cout << "\n__cplusplus 值: " << __cplusplus << endl;
    if (__cplusplus >= 201103L) {
        cout << "当前已启用 C++11 或更高标准!" << endl;
    }

    return 0;
}
```

**预期输出：**

```
auto 推导出类型: 42
范围 for 循环遍历: 10 20 30 40 50
to_string 结果: 3.141590
stod 结果: 2.71828

__cplusplus 值: 201103
当前已启用 C++11 或更高标准!
```

> **注意**：`__cplusplus` 宏在 C++11 标准下定义为 `201103L`，C++14 为 `201402L`，C++17 为 `201703L`。可以通过检查这个宏来判断当前使用的 C++ 标准版本。

## 6. 小结

C++11 是 C++ 语言发展史上的分水岭。它引入了大量现代化特性，让 C++ 编程变得更安全、更高效、更富表现力。从本章开始，我们将逐一深入学习这些特性。掌握了 C++11，你就迈入了"Modern C++"的大门。
