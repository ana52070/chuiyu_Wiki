# 2026-04-06

## 今晚主题

Plan 模式为什么能让 Claude Code 少返工。

## 今晚发出的内容

今晚接前两天的 `CLAUDE.md` 往下走，讲一个特别实用的习惯：**复杂任务别急着让 Claude Code 直接改代码，先开 Plan 模式，让它只想方案、不动文件。** 这样你能先看它准备改哪些地方、为什么这么改、有没有漏掉边界条件，方向对了再放它执行，整体反而更省时间。

比如你想让它“重构登录流程”，如果一上来就开干，常常会边改边偏；但先让它列出：要动哪些文件、登录态怎么流转、风险点在哪，你只要先把路线纠正好，后面执行就顺很多。

今晚记住一句话：**复杂任务先想清楚再动手，通常比边做边救火更快。**

## 下一次可衔接方向

Plan 模式和后续执行怎么配合更顺：为什么方案确认后再开 Auto 或批量执行，往往更丝滑。
