# 2026-04-05

## 今晚主题

好的 `CLAUDE.md` 该写什么，不该写什么。

## 今晚发出的内容

今晚接昨天继续往下走，记一个特别实用的点：**`CLAUDE.md` 不是写得越多越好，而是要只留下“高频约束”和“真实踩坑”**。像项目用什么启动命令、提交前必须跑什么检查、哪些目录别乱动、组件风格怎么统一，这些都值得写；但每个函数怎么用、每个文件做什么，别全塞进去，不然 Claude 还没开始干活，上下文就先被说明书占满了。

比如你常常提醒它：“改完前端先跑测试，别碰 `scripts/legacy/` 里的旧脚本。”这种规则写进去就很值；但“某个 util 里每个参数的细节”通常更该留在代码和文档里。

今晚记住一句话：**`CLAUDE.md` 写的是护栏，不是百科全书。**
