# 2026-04-04

## 今晚主题

为什么 `CLAUDE.md` 像给 AI 的地图。

## 今晚发出的内容

今晚接着昨天往下走，记一个特别关键的点：**`CLAUDE.md` 本质上不是文档，而是你提前写给 Claude Code 的“项目内工作说明”**。它会在新会话开始时先读这个文件，所以里面放的不是大而全的百科，而是“这个项目最容易踩坑的规则”。

比如你在一个项目里总要求它：先跑测试再提交、前端统一用某套组件、别乱改某个配置文件。那这些话如果每次都重新讲，很浪费；写进 `CLAUDE.md` 后，它一进项目就更像已经被你带过一遍的新同事。

今晚记住一句话：**`CLAUDE.md` 不是说明书，而是让 Claude Code 少走弯路的项目护栏。**

## 下一次可衔接方向

好的 `CLAUDE.md` 到底该写什么、不该写什么。
